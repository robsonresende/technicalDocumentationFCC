add

git add

Add file contents to the index

OPTIONS

  • -n - Don’t actually add the file(s), just show if they exist and/or will be ignored.
  • -b - Be verbose.
  • -f - Allow adding otherwise ignored files.
  • -i - Add modified contents in the working tree interactively to the index.
  • -p - Interactively choose hunks of patch between the index and the work tree and add them to the index.
status

git status

Show the working tree status

OPTIONS

  • -s - Give the output in the short-format.
  • -b - Show the branch and tracking info even in short-format.
  • --show-stash - Show the number of entries currently stashed away.
  • --long - Give the output in the long-format. This is the default.

diff

git diff

Show changes between commits, commit and working tree, etc

OPTIONS

  • --patch - Generate patch (see section on generating patches). This is the default.
  • --no-patch - Suppress diff output.
  • --unified - Generate diffs with 'n' lines of context instead of the usual three.
  • --output - Output to a specific file instead of stdout.
  • --raw - Generate the diff in raw format.
commit

git commit

Record changes to the repository

OPTIONS

  • -a - Tell the command to automatically stage files that have been modified and deleted, but new files you have not told Git about are not affected.
  • -p - Use the interactive patch selection interface to chose which changes to commit. See git-add[1] for details.
  • -c - Like -C, but with -c the editor is invoked, so that the user can further edit the commit message.
  • -m - Use the given 'msg' as the commit message.
reset

git reset

Reset current HEAD to the specified state

OPTIONS

  • -q - Be quiet, only report errors.
  • --pathspec-from-file - Pathspec is passed in 'file' instead of commandline args.
  • --pathspec-file-nul - Only meaningful with --pathspec-from-file.
  • -- - Do not interpret any more arguments as options.
  • pathspec - Limits the paths affected by the operation.
